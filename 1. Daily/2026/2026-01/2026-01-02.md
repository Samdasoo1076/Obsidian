목표:
브라우저 즐겨찾기(Bookmarks)를 보완하는 Chrome 확장 프로그램을 개발한다.
단순 URL 저장이 아니라, "왜 저장했는지"와 "페이지의 정확한 위치(DOM)"를 함께 기록하고,
나중에 다시 접속했을 때 자동으로 해당 위치로 이동하고 메모를 표시하는 것이 목적이다.

핵심 기능 요구사항:

1. 즐겨찾기 저장 방식
- Chrome 기본 북마크 버튼을 직접 가로채는 것은 불가능하므로,
  확장 프로그램이 자체적으로 "북마크 감지/연동" 방식으로 동작해야 한다.
- 사용자가 페이지에 진입했을 때:
  - 현재 URL이 Chrome 북마크에 존재하는지 chrome.bookmarks API로 확인
  - 존재하고, 확장 프로그램에 저장된 메타데이터가 있다면 자동 동작

2. 위치(DOM) 저장
- 사용자가 페이지 내 특정 위치에서 "이 위치 저장" 버튼을 누르면:
  - 클릭 시점의 DOM 정보를 저장
    - 우선순위:
      1) data-id / id
      2) 안정적인 CSS selector
      3) XPath (fallback)
  - 해당 위치의 텍스트 일부 (context)도 함께 저장 (DOM 변경 대비)

3. 메모 저장
- 북마크 단위로 다음 정보를 저장:
  - URL
  - 페이지 title
  - DOM selector / XPath
  - 텍스트 context
  - 사용자 메모 (string)
  - 저장 시각

4. 페이지 재방문 시 동작
- 사용자가 북마크로 페이지 진입 시:
  - DOMContentLoaded 이후 저장된 selector 탐색
  - DOM이 존재하면:
    - 해당 위치로 자동 스크롤
    - 하이라이트 표시
    - 메모를 작은 floating panel 또는 tooltip으로 표시
  - selector가 깨졌을 경우:
    - 저장된 텍스트 context 기반으로 유사 DOM 탐색 시도

5. UI 요구사항
- Content Script에서 작은 버튼 제공:
  - "이 위치 저장"
- 저장 후:
  - 메모 입력 모달 표시
- 재방문 시:
  - 화면 우측 하단 또는 해당 위치 근처에 메모 UI 표시

6. 기술 스택 제약
- Chrome Extension Manifest V3
- 언어: TypeScript
- 구조:
  - background service worker
  - content script
  - popup (선택)
- 저장소:
  - 1차: chrome.storage.local
  - (확장 고려) 서버 연동 가능하도록 구조 분리

7. 제한 사항 인지
- Chrome 기본 북마크 버튼 클릭 이벤트는 직접 가로챌 수 없음을 전제로 설계
- 북마크 연동은 chrome.bookmarks API 기반으로 구현

8. 산출물
- 프로젝트 폴더 구조
- manifest.json
- 주요 TypeScript 파일
- DOM selector 안정화 전략 설명
- 북마크 연동 로직 설명

이 확장 프로그램은 "기억 보조용 북마크"가 목적이며,
사용자가 나중에 다시 왔을 때
"아 이거 왜 저장했지?"를 해결해주는 UX를 최우선으로 한다.
