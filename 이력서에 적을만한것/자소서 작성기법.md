## STAR 기법이란?

경험을 설명할 때 사용하는 4단계 프레임워크입니다.

- **S (Situation)**: 어떤 문제가 있었나?
- **T (Task)**: 내가 해결해야 할 목표는?
- **A (Action)**: 구체적으로 무엇을 했나?
- **R (Result)**: 측정 가능한 성과는?

## Before vs After

### ❌ Before

> 저는 팀 프로젝트에서 리더 역할을 맡아 적극적으로 의견을 조율하고 문제를 해결했습니다. 어려움이 있었지만 포기하지 않고 끝까지 최선을 다했으며, 그 결과 좋은 성과를 냈습니다.

**문제점**: 추상적 형용사, 구체성 제로, 숫자 없음

### ✅ After

> **[S]** 4인 팀 프로젝트 중간에 백엔드 담당자 1명이 이탈하면서 API 개발이 2주 지연되었습니다.  
> **[T]** 프론트엔드 담당이었지만, 데모 데드라인(3주 후)을 맞추기 위해 백엔드 일부를 병행하기로 결정했습니다.  
> **[A]** Node.js 공식 문서와 팀원이 남긴 코드를 3일간 학습한 뒤, REST API 4개 중 2개를 구현했습니다. 나머지는 Mock 데이터로 대체해 프론트 작업을 먼저 완료하고, 팀원과 매일 30분 코드 리뷰를 진행했습니다.  
> **[R]** 데모 1일 전 전체 기능 통합 완료. A+ 학점 획득 후 Node.js 기초를 익혀 풀스택 프로젝트 1건 추가 진행.


### S (Situation) - 문제 상황

```null
"[프로젝트/업무] 중 [문제]가 발생했습니다."
"[지표]가 [숫자]에 달해 [부정적 영향]이 예상되었습니다."
```

**예시**

- "배포 후 사용자 이탈률이 12%까지 증가했습니다."
- "레거시 코드 의존성 문제로 신규 기능 추가 시간이 평균 5일 소요되었습니다."

### T (Task) - 목표

```null
"[역할]로서 [목표]를 달성해야 했습니다."
"[기간] 내에 [지표]를 [목표치]로 개선하는 것이 과제였습니다."
```

**예시**

- "프론트엔드 담당으로 로딩 시간을 3초 이내로 줄여야 했습니다."
- "2주 내 결제 성공률을 95% 이상으로 회복시키는 것이 목표였습니다."

### A (Action) - 행동 ⭐ 가장 중요!

```null
"[도구/방법]를 사용해 [원인]을 분석했습니다."
"[기술/프로세스]를 도입해 [개선 작업]을 진행했습니다."
"[빈도/기간] 동안 [활동]을 수행했습니다."
```

**예시**

- "Chrome DevTools로 렌더링 병목을 분석한 결과, 초기 번들 사이즈(2.3MB)가 원인임을 확인했습니다."
- "Code Splitting과 Lazy Loading을 적용해 초기 로드 파일을 640KB로 감소시켰습니다."
- "주 2회 팀 회고를 통해 병목 구간을 지속적으로 파악하고 개선했습니다."

**⚠️ Action은 전체의 50% 이상 비중을 차지해야 합니다!**

### R (Result) - 결과

```null
"[지표]가 [Before] → [After] ([변화율])로 개선되었습니다."
"그 결과 [부가 효과]를 달성했습니다."
```

**예시**

- "페이지 로딩 시간 3.8초 → 1.2초 (-68%), 이탈률 12% → 5%로 감소했습니다."
- "이후 동일한 최적화 기법을 다른 3개 페이지에 적용했습니다."



## 흔한 실수 5가지

### 1. S와 T를 구분 못 함

- ❌ "API 성능 개선이 필요했습니다."
- ✅ S: "API 응답이 2.5초로 느려 타임아웃 발생" / T: "1주일 내 1초 이내로 개선"

### 2. Action에서 "We" 사용

- ❌ "우리 팀은 Redis를 도입했습니다."
- ✅ "제가 Redis 캐싱 구조를 설계하고, 팀원 2명과 구현했습니다."

### 3. Result에 숫자 없음

- ❌ "성능이 크게 개선되었습니다."
- ✅ "응답 시간 2.5초 → 0.4초 (-84%)"

### 4. Action이 너무 짧음

Action은 전체의 **50% 이상** 비중을 차지해야 합니다.

### 5. 모든 경험을 STAR로 쓰려고 함

STAR는 **문제 해결 경험**에만 씁니다. 단순 학습이나 참여는 1-2줄로 간단히.


## STAR 적용하기 좋은 경험

### ✅ STAR 필수

- 문제 해결 경험 (성능, 버그, 장애)
- 프로젝트 경험 (목표 → 구현 → 결과)
- 최적화/개선 (Before/After 비교 가능)
- 협업 갈등 해결
- 실패 후 재도전

### ❌ 일반 서술로 충분

- 단순 학습 (강의 수강, 책 읽기)
- 단순 참여 (해커톤, 스터디)
- 자격증 취득
- 도구 사용 경험
- 일상적 루틴 업무
